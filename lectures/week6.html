This week is going to be an open lab with a structured assignment. This means you can follow along with the lecture as we build out a template, or you can forge ahead on your own with the below-listed requirements for your template if you like.

What we will build

Today, we will build on what we did last week and expand our template to include a lot of extra information, a more flexible layout and user customization.

While the syntax and UI are different for every CMS (Content Management System), the concept is identical. Whether it be Wordpress, Drupal, Joomla, Jekyll, Shopify, OpenCart, PrestaShop, etc., all of them have templates that you develop with HTML, CSS, JS and whatever Syntax the application uses.

Today, on our Tumblr template, we will create the following:

  • HTML, CSS used for the code to mark up content as best as possible
  • Post Metadata ensuring each post is readily viewable
  • Header with account details and search
  • Footer with contact and social links
  • Pages for more static content
  • Tags on each post
  • Pagination for when you have more posts than fit on one page
  • User Customization which gives some control of the look and layout to the user
  • Layout of the page using a grid tool
  • Asset Management for fast access to scripts and styles

Almost all of what we are building can be found on the Create a Custom HTML Theme documentation page.

HTML, CSS

Everything from the doctype to the various tags (like header, footer, nav, figure, article, section) that make a document sematically rich should be used whenever possible.

The same goes for CSS, using Elements, Classes and IDs can easily be done with your template. Utilizing syntax to make intelligent decisions about how to style a particular part of your code is a mark of a sophisticated template.

Pages

Pages are a way of creating more static pages, such as About, Contact, Location, etc. This is content that isn’t chronological or “bloggy”.

Creating pages is also done from the Edit Theme panel, near the bottom where it says Add Page.

While Custom Layout and Redirect are options, we will use Standard Layout to create at least 3 pages. Once those pages are created, you’ll need to link to them somehow.

The documentation will help with this, but a few things to bear in mind:

  • Use an UL for the navigation, with LI tags for each page
  • Use a NAV tag to extend your HTML5 semantic richness on the page

Tags

Tags can be added to any/all posts as a secondary way of searching or sorting your content. This concept is identical to hashtags used in sites like Flickr, Instagram, Twitter, Facebook. Click the tag and the system will look for all instances of that tag within your site and show them on a page.

For now, you will need to go back to your dashboard and add tags to each of your posts. Choose 2 or 3 tags that describe each post.

Once you have tagged all your content, you can build the code that supports viewing the tags within the post metadata.

  • Add tags to your posts
  • Add code to support tags in your post metadata
  • Create a link to each tag that opens a tag page

Pagination

Once you have enough posts, you will eventually have too many to show on a single page. At that point, you will want to allow users to navigate through your site by viewing older (or newer) posts. The pagination system within Tumblr is well developed and easy to implement.

A great example of pagination that everyone has seen is when you search for something on Google, and at the bottom of the page, you will see the Gooooooooogle UI with Next and Previous available when required.

  • Create NEXT and PREVIOUS links only on pages that need them
  • Create jump pagination that displays what page the user is on and how many pages are available
