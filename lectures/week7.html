Template Final

The main thing I want to share is the Asset Management part of Tumblr which is similar for all Content Management Systems.

Instead of managing files on external servers or embedding scripts and CSS in your HTML, you can use asset management to make it a lot easier.

Benefits are:

  • Reusable code (familiar for CSS and script files anyway)
  • Cached for faster access
  • Can also utilize minified and gzipped content for even faster load times

Javascript & jQuery

jQuery is a Javascript Framework that has become almost ubiquitous in front end development. After starting as a last-minute coding exercise by a Netscape employee, it is now the most commonly used programming language in the world.

However, as a scripting language, it is sometimes as much art as it is science.

What is JavaScript?

From Wikipedia:

JavaScript is a scripting language most often used for client-side web development. JavaScript was influenced by many languages and was designed to have a similar look to Java, but be easier for non-programmers to work with.

JavaScript is easy - right?

For many web developers, JavaScript is another language that they need to learn, one that isn’t as straightforward as the markup and presentation of HTML and CSS.

Many people simply steal and re-use rather than coding their own. Javascript was also a language that was poorly supported for years, adding to the difficulty.

Other Libraries & Frameworks

  • Prototype - one of the first libraries, still very good
  • script.aculo.us - One of the best effects libraries, uses Prototype Lirary
  • Mootools - a more recent player with some excellent UI effects

jQuery

  • jQuery is a JavaScript Library
  • Designed especially for making JS manipulations easier
  • Fairly small footprint (smaller than an average image)
  • Gives access to any DOM element
  • If you can access the element from CSS (level 1-3), you can access it from jQuery
  • Very little requirement to change any HTML
  • Syntax is a little scary at first, but easy to pick up
  • The main reason I chose it for this course - simpler syntax, full CSS accessibility

jQuery Requirements

  • Inclusion of the jQuery code in your HTML:
  • Download from jQuery
  • Three main forms:
  • Uncompressed - best for reviewing the code
  • Packed - most often used for production systems
  • Minified - used for systems with Unix backend (requires GZip)
  • Or, can be done directly from jQuery
  • Need to include your own code either as an embedded block or linked to an external file
  • Basic functions are very easy to write
  • Works in almost all browsers (including IE6! - Mostly)

Using the Console & jQuerify

One of the ways you can test how jQuery will work on a page (or, more importantly, if a selector will work) is to have jQuery on a page and use the Developer Console to pass requests in.

This works best on Firefox or Chrome (I will use Firefox).

You will need to create a bookmarklet from jQuerify, which is used to insert jQuery to a page that might not already have it. It also works as a great tool for identifying what version of jQuery is currently being used.
